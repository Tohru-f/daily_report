## わかったこと
- git add * とすれば、カレントディレクトリ内のファイルなどを全て対象にできる。但し、カレントディレクトリは含まない。特定のディレクトリだけをコミットしたい場合はディレクトリを移動して、git add * すれば一度に必要なファイル類を全てコミットできる
- git rm コマンドは gid add と一緒でファイルをステージングする機能があるので、その後はgit commitコマンドを実行すれば良い。git rm の後に git add を実行する必要はない。
- TypeScriptのコードをgithubにデプロイする場合、コンパイルしたコードをPushする必要がある。また、Github Pagesではルートディレクトリ、またはdocsディレクトリ直下にindex.htmlを初めとするファイル類を置かないとブラウザのコンソールエラーでファイルが見つからないと言われてしまう。
